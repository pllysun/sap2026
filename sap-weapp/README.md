# 软协课表 · 微信小程序（sap-weapp）

复刻安卓 App（`sap-android`）的微信原生小程序版。后端复用 `sap-backend` 的现成 REST API，
小程序只是另一个 HTTP 客户端。

## 技术栈
- 微信原生（WXML / WXSS / JS），无第三方框架/组件库。
- 后端：`sap-backend`（Spring Boot + Sa-Token），教务数据由后端代抓（WebVPN→CAS→强智）。

## 目录结构
```
config.js            全局配置（BASE_URL、storage key、会员角色阈值）
app.js / app.json / app.wxss   应用壳、页面注册、tabBar、全局样式与 CSS 变量
utils/
  request.js   wx.request 封装（注入 sap-token 头、解包 Result、登录失效跳转）
  api.js       后端接口（/api/auth、/api/jw）
  store.js     本地状态（会员/用户/当前学号/多账号多学期课表缓存/成绩考试缓存）
  schedule.js  课表合并与网格布局（教务课+自建课、按周过滤、重叠分栏、节次定位）
  week.js / periods.js / term.js / color.js / gradeStats.js  纯逻辑移植自安卓
pages/
  login/       账号密码登录（协会会员账号）
  bind/        绑定教务账号（含图形验证码 / 短信 MFA 分阶段）
  schedule/    课表（核心：多账号/多学期、横滑翻周、自建课、详情、设置入口）
  grade/       成绩 / 考试 / 评教（分段切换 + 防风控缓存 + 手动同步）
  profile/     我的（用户信息、教务账号管理、设置入口）
  settings/    设置（课表显示设置、主题色、隐私协议、关于、退出登录）
  editCourse/  新增/编辑自建课程
```

## 如何运行
1. 启动后端 `sap-backend`（默认 `:8081`）。
2. 微信开发者工具「导入项目」选本目录；AppID 可用测试号。
3. 「详情 → 本地设置」勾选 **不校验合法域名**（开发期）。
4. `config.js` 的 `BASE_URL` 改为后端地址（本机 `http://localhost:8081` 或局域网 IP）。
5. 编译运行 → 登录（协会会员账号）→ 课表页绑定教务账号 → 查看课表/成绩/考试/评教。

## 已实现
- 账号密码登录 + token 免登（`sap-token` 头，storage 持久化，失效自动回登录）。
- 绑定教务（图形验证码 base64 图 + 短信 MFA 续验，分阶段）。
- 课表：多教务账号切换、多学期切换、横滑翻周、今日高亮、节次时间列、自建课增删改、
  同格重叠分栏、非本周灰显、备注（无固定时间课）、网格设置（节数/高度/字号/周末/总周数/开学日期）。
- 成绩：GPA/均分/学分等概览、分数段分布条、按学期分组列表（统计逻辑 1:1 移植自安卓）。
- 考试：按学期查询、考场/座位展示。
- 评教：列表 + 一键自动评教。
- 我的：用户信息、教务账号管理（切换/解绑/新增）、设置。
- 主题色切换、隐私协议、关于、退出登录。
- 防风控：成绩/考试/评教进页面只读缓存，刷新靠手动同步。

## 未实现 / 降级（受微信小程序平台约束，非代码缺陷）
- **WebView 端上抓取模式**：小程序 `web-view` 沙箱无法注入 JS / 读 DOM / 配第三方域名 → 整条作废。
  小程序统一走后端代抓（已按此决策实现）。
- **桌面小组件**：小程序无此能力。
- **上课提醒**：无本地定时通知；后续可用微信「订阅消息」+ 后端定时推送替代（待做）。
- **课表分享图 / 应用内升级**：分享图可后续用 `canvas 2d` 重画；升级由微信热更新接管，无需自做。

## 上线前待办（重要）
- **HTTPS + ICP 备案域名**：发布/真机体验时 `wx.request`、`uploadFile` 域名必须是已备案 HTTPS，
  并加入小程序后台「服务器域名」白名单。
- **微信登录**（可选）：当前沿用协会账号密码登录；如需 `wx.login` 静默登录，后端补 openid↔会员绑定。
- **审核合规**：教务代抓 + 收集学号密码在微信审核敏感，需提前评估（可能需学校授权/资质）。
